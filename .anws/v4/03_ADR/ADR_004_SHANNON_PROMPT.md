# ADR-004: 香农信息论提示词工程

**状态**: Accepted
**日期**: 2026-05-22
**决策者**: ZeroSearch v0.4 genesis
**影响范围**: Shannon Strategy Skill, Search Execution Skill

---

## 背景

v0.3 的搜索本质是"将用户查询原样传给 Google AI Mode"。用户输入 "React 性能优化怎么搞"，就原样搜索 "React 性能优化怎么搞"。这种关键词匹配思维是上一代搜索引擎（10 个蓝色链接）的产物。

Google AI Mode（udm=50）是一个 AI 驱动的搜索引擎，能够理解搜索意图并进行语义级别的综合。但我们没有充分利用这一能力——含有大量低信息量通用词（"怎么""搞"）的查询，削弱了 AI 的理解精度。

核心理念来自香农信息论：
```
I(x) = -log₂P(x)
```
- P(x) 越小（词越罕见），I(x) 越大（信息量越高）
- 高信息量关键词 = 更精准的搜索结果

## 核心原则

**关键词不怕错，就怕太普通。**

**数学本质**：信息量 = -log₂(概率)，概率越低的词，信息量越高，搜索越精准。

---

## 香农信息论速查

**自信息**（单个关键词的信息量）：
```
I(x) = -log₂P(x)
```
- P(x) 越小 → I(x) 越大 → 关键词信息量越高
- 示例：P("马都工")=0.0001 → I=13.3 bit；P("马前卒")=0.01 → I=6.6 bit

**联合信息**（组合关键词的信息量）：
```
I(x,y) = I(x) + I(y)   （当 x,y 独立时）
```
- 组合多个独立关键词，信息量相加
- 示例：I("山楂糕")+I("枣泥饼")+I("酸枣冰棍") = 10+10+10 = 30 bit

**条件熵**（已知证据后目标的剩余不确定性）：
```
H(目标|证据) = H(目标) - I(目标;证据)
```
- 每轮搜索提供新证据，降低条件熵
- 终止条件：H(目标|证据) ≈ 0

---

## 迭代搜索法（贝叶斯更新）

用贝叶斯更新引导迭代，逐步降低条件熵，直到信息充分。

```
初始状态：H(目标) = 高（不确定性高）

每轮搜索：
├─ 用高信息量关键词搜索：max I(关键词)
├─ 阅读结果，更新先验：P(目标|新证据)
├─ 根据新发现调整下一轮关键词
└─ 条件熵 H(目标|已有证据) 逐步降低

终止条件：H(目标|已有证据) ≈ 0（信息充分）
```

**语言选择**：查询语言应与目标内容语言匹配

## 关键词设计速查

### 高信息量（优先使用）

| 类型 | 示例 | 信息量 |
|------|------|--------|
| 具体公式 | "I(x) = -log p(x)" | 极高（P极低） |
| 专业术语 | "TF-IDF"、"互信息" | 高 |
| 独特表述 | "马都工"、"山楂糕枣泥饼" | 高 |
| 具体数字 | "15个人诱导40%人群" | 高 |
| 型号/版本 | "React 19 Server Components" | 高 |

### 低信息量（避免使用）

| 类型 | 示例 | 信息量 |
|------|------|--------|
| 通用词 | "信息"、"方法"、"怎么" | 极低（P极高） |
| 过于宽泛 | "信息论"、"React" | 低 |
| 同义词重复 | "Shannon+香农" | 组合增益≈0 |

**公式**：I(关键词) = -log₂P(关键词)，选择 P 最小的词

---

## 组合搜索技巧

当单个词信息量不足时，用组合：

```
失败：山楂糕 → 海量广告
成功：山楂糕 + 枣泥饼 + 酸枣冰棍 → 精准命中
```

**公式**：I(x,y,z) = I(x) + I(y) + I(z)（独立关键词信息量相加）

**组合原则**：
1. 选择不同维度的词（避免同义词，确保独立性）
2. 选择不常见的组合（提高不可预测性）
3. 3-5 个词为宜

---

## 迭代优化策略

### 关键词调整

每轮搜索后，根据结果更新关键词：

```
Round 1：I(关键词1) → 结果1 → H(目标|结果1) 降低
Round 2：I(关键词2) → 结果2 → H(目标|结果1,结果2) 更低
Round 3：I(关键词3) → 结果3 → H(目标|结果1,2,3) ≈ 0
```

### 终止条件

- **信息充分**：H(目标|已有证据) ≈ 0，新轮次不再有增量信息
- **判断标准**：连续两轮搜索发现高度重复内容，说明已收敛

---

## 语言匹配原则

**查询语言应与目标内容语言匹配**

| 目标内容 | 推荐语言 | 原因 |
|---------|---------|------|
| 日本内容 | 日语 | 找到日语语料，更深入 |
| 英文内容 | 英文 | 找到国际视角，更广泛 |
| 中文内容 | 中文 | 找到中文语料，更精准 |
| 混合内容 | 双语各一轮 | 覆盖全面 |

---

## 容错性

高质量关键词即使有偏差，效果也优于低质量关键词：

- 搜"马都工"（正确）→ 全部精准结果
- 搜"马杜工"（输入错误）→ 仍然精准
- 搜"马前卒"（通用词）→ 大量无关结果

**公式解释**：I("马都工") >> I("马前卒")，即使有偏差，I("马杜工") 仍然 >> I("马前卒")

**启示**：先把方向打对，细节可以后续修正。

---

## 决策

**采用香农信息论指导的提示词工程，作为搜索查询构造的核心策略。**

### 方案对比

| 维度 | A: 原样搜索 (v0.3) | B: 香农提示词工程 (选择) | C: 自动多轮迭代 |
|------|:--:|:--:|:--:|
| 搜索精准度 | ⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 复杂度 | 极低 | 低（纯文本 Skill） | ❌ 高（多轮编排） |
| AI 上下文消耗 | 低 | 中（Skill 指导） | ❌ 高 |
| 用户控制 | 无 | ✅ 用户手动迭代 | ❌ 自动不可控 |
| 实现成本 | 零 | 低（Markdown Skill） | ❌ 高（多轮 + 状态管理） |

### 核心策略

Shannon Strategy Skill 指导 Claude 执行以下查询优化流程：

1. **意图提取**: 从用户查询中提取核心搜索意图
2. **关键词筛选**: 去除通用词（"怎么""方法""优化"），保留高信息量词（专业术语、版本号、具体数字）
3. **维度组合**: 选择独立维度词组合（I(x,y,z) = I(x)+I(y)+I(z)）
4. **语言匹配**: 目标内容语言 = 查询语言（日本内容用日语、英文文档用英语）
5. **容错思维**: 高信息量关键词即使有偏差，效果仍优于精准低信息量词

示例转换:
```
输入: "React 19 Server Components 怎么优化性能"
输出: "React 19 Server Components streaming SSR partial prerendering performance"
```

## 影响分析

### 正面影响
- 搜索精准度显著提升：去除噪声词，保留高信息量信号
- 零新依赖：纯 Markdown Skill，不需要额外的 NLP 库或 API
- 可迭代优化：Skill 文件修改即可升级策略，不需要改代码
- 用户可控：AI 建议更优关键词，用户决定是否追轮

### 负面影响
- 依赖 Claude 的理解能力：如果 Claude 不理解领域，可能选择错误的关键词
- Skill 内容增加：相比 v0.3（无策略指导），探索策略 Skill 约 100-150 行

### 缓解措施
- Shannon Strategy Skill 包含详细的关键词评估矩阵和示例
- Skill 提供容错指导：宁可高信息量有偏差，不要低信息量精准
- 实践验证：基于快速搜索策略.md 中的实战验证案例
